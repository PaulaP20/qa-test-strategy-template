# Bug Report Template

## Summary
Short title describing the issue.

Example:
Login button does not respond after entering valid credentials.

---

## Description
Detailed explanation of the issue.

Include relevant context, system behavior, and any observations that may help developers understand the problem.

Example:
After entering valid credentials and clicking the login button, the application does not respond. No error message is displayed, and the user remains on the login page.

Specify where the issue was found.

- Environment: (Development / Staging / Production)
- Browser / Device:
- OS:
- Application version:

---

## Preconditions
List any conditions required before reproducing the issue.

Example:
User account must already exist.

---

## Steps to Reproduce

1. Navigate to the login page
2. Enter valid credentials
3. Click the "Login" button

---

## Expected Result
Describe what should happen.

Example:
User should be successfully authenticated and redirected to the dashboard.

---

## Actual Result
Describe what actually happens.

Example:
Nothing happens when clicking the login button.

---

## Severity

Impact of the defect on the system.

- Blocker – System cannot continue, or critical functionality is completely unavailable.
- Critical – Core functionality broken with no reasonable workaround.
- Major – Important functionality affected, but a workaround may exist.
- Minor – Small issue with limited impact on functionality.
- Trivial – Cosmetic issues or very small problems that do not affect functionality.
- Enhancement – Suggested improvement or feature request rather than a defect.

Note: Enhancements represent product improvement suggestions rather than defects.
